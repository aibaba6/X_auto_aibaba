# 運用ルール（自動収集 -> 予測投稿 -> 引用RT）

対象アカウント: `@AbaboDesign`

## 1. 収集ルール

- 実行頻度: 1日3回（07:00 / 12:00 / 18:00）
- 収集上限: 1回あたり20件
- 情報源:
  - AI公式ニュース（OpenAI等）
  - デザイン/UXメディア
  - X検索（AI x デザイン）
- 除外キーワード: `PR`, `広告`, `Giveaway`, `無料配布`, `副業`

## 2. 投稿生成ルール（通常投稿）

- 1回の通常投稿数: 最大1本
- 文体: 冷静さを保ちつつ、僅かにゆるいカジュアルさを含める
- 投稿テンプレート（必須4要素）:
  - 事実: 最新情報を1文で要約
  - 示唆: デザイン実務への影響を1文
  - 予測: 3-6ヶ月で起こる変化を1文
  - 行動: 今週やる実験を1文
- 本文では `事実:` `示唆:` のようなラベルは使わない（自然文で書く）
- 可読性のため改行を入れる（2-4段落推奨）
- 箇条書きは1-3項目まで（必要時のみ）
- 記号や中黒（・）で読みやすくしてよい
- 絵文字は0-2個まで
- 文字数: 120-280字
- ハッシュタグ: 2-3個（文末）
- 禁止表現: `絶対`, `100%`, `必ず`

## 2.2 時間帯ごとの投稿ルール（必須）

- 朝 07:00 (`morning`):
  - 目的: 勉強になる、有益、保存したくなる投稿
  - 文字数目安: 170-280字（ボリューム多め）
  - 引用RT: なし
  - テーマ: デザイン基礎/応用/豆知識の普遍テーマを優先（週内で内容重複を避ける）
- 昼 12:00 (`noon`):
  - 目的: サクッと読める要点 + 最新AI情報のシェア
  - 文字数目安: 90-180字（短め）
  - 引用RT: あり（最新情報の引用を優先）
  - 候補選定: `動画付き > 画像付き > テキスト`
  - コメント: URLを入れない（リンクによるインプレ低下を回避）
  - 投稿文作成: 事前固定しない。投稿予定の約30分前に再収集して生成（鮮度優先）
  - 自動ジョブ: 5分ごとに `refresh-noon-queue` を実行し、対象noonキューを上書き
- 夕 18:00 (`evening`):
  - 目的: 共感ベースで一日の学びを着地させる投稿
  - 文字数目安: 110-220字
  - 引用RT: あり

## 2.1 曜日ごとの投稿テーマ配分

- 月: 速報要約（先週末〜当日の主要AIニュースを要点整理）
- 火: 実装/制作（AIをデザイン実務に組み込む具体手順）
- 水: 比較検証（複数ツールや手法の使い分け）
- 木: 予測検証（以前の予測の答え合わせと修正）
- 金: 戦略視点（3-6ヶ月の変化と勝ち筋）
- 土: ケース分析（良い事例/悪い事例の分解）
- 日: 週間まとめ（来週の実験計画と観測ポイント）

備考:
- ツールは実行日の曜日を自動判定し、その日のテーマを生成プロンプトに反映する。
- 投稿は毎回 `事実 / 示唆 / 予測 / 行動` の4要素を維持する。

## 3. 品質ゲート（投稿前）

下記を満たさない投稿案は破棄する。

- 予測キーワードを1つ以上含む
  - 例: `予測`, `起こる`, `3ヶ月`, `6ヶ月`
- 行動キーワードを1つ以上含む
  - 例: `今週`, `試す`, `検証`, `実験`
- 禁止表現を含まない
- 文字数・ハッシュタグ上限を満たす

## 4. 引用RT候補選定ルール

- 1回の引用投稿数: 最大1本
- 候補の除外条件:
  - 40字未満 / 240字超
  - 先頭が `@`（返信投稿）
  - （必要に応じて）URL含有投稿
- 優先キーワード:
  - `なぜ`, `課題`, `失敗`, `実験`, `検証`
- 採用基準:
  - スコア3以上を採用

## 5. 引用コメント生成ルール

- 80-180字
- 相手の主張を要約しつつ敬意を示す
- 追加価値を1つ入れる
  - 補足視点 or 反証可能な予測
- 感想のみで終わらず、実務アクションを示す

## 6. 運用手順

1. `dry_run: true` で1週間ログ検証
2. 破棄理由（`[DROP DRAFT]`）を見てキーワードを調整
3. `posting.force_post_if_no_passed: true` で投稿欠損を防止
4. 問題なければ `dry_run: false` に切替
5. 週1回、予測の答え合わせ投稿を手動で実施

## 7. ガードレール

- 医療/法律/金融の断定助言をしない
- 未確認情報は断定しない
- 個人や企業への誹謗・断定批判をしない
- 引用元の文脈を歪めない
